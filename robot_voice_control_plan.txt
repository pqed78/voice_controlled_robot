[수정된 상세 계획서: ROS2 Humble & Isaac Sim 5.1 기반 음성 제어 로봇]

1. 시스템 환경 및 제약 조건
  - OS/Middleware: Ubuntu 22.04 + ROS2 Humble
  - Simulator: NVIDIA Isaac Sim 5.1
  - GPU: 24GB VRAM (Isaac Sim과 LLM 병행 구동 최적화 필요)
  - LLM 모델: Ollama 기반 Llama 3.1 8B (4-bit Quantized) 또는 Phi-3.5 (Mini)

2. 시스템 아키텍처 (VRAM 효율적 설계)
  가. 음성 입력 (STT): Faster-Whisper (Small 모델 사용, GPU 점유 최소화)
  나. 지능형 파싱 (Ollama): 
      - Llama 3.1 8B 혹은 Phi-3.5를 사용하여 자연어를 JSON 액션으로 변환.
      - 시스템 프롬프트 최적화를 통해 파싱 정확도 향상.
  다. 사용자 확인 프로세스 (TTS): Piper (CPU 기반 구동 가능하여 VRAM 절약)
      - 로봇: "거실로 이동하여 오른팔로 컵을 잡을까요?" -> 사용자 음성 확인 대기.
  라. 로봇 제어 브릿지 (ROS2 Node):
      - Ollama의 JSON 출력을 분석하여 ROS2 Action(Nav2, MoveIt2)으로 전달.

3. Isaac Sim 5.1 연동 계획
  - Robot Model: URDF/Xacro 기반 양팔 차동 로봇 (Differential Drive + Dual Arm)
  - Controller: ROS2 Control (Diff Drive Controller, Joint Trajectory Controller)
  - Perception: Isaac Sim 내의 센서(Lidar, Camera) 데이터를 ROS2로 퍼블리시하여 실시간 네비게이션 및 매니퓰레이션 수행.

4. 세부 구현 단계
  Step 1: Ollama 전용 모델 파일(Modelfile) 작성
    - 로봇의 액션 정의(move_to, pick, place, stop 등)를 시스템 프롬프트로 고정.
  Step 2: ROS2-Ollama 파이썬 노드 개발
    - 음성 텍스트 수신 -> Ollama API 호출 -> JSON 파싱 -> 사용자 확인 로직 구현.
  Step 3: Isaac Sim 5.1 로봇 시뮬레이션 환경 구축
    - 차동 구동부(Nav2) 및 양팔(MoveIt2) 설정.
    - `isaac_ros_bridge`를 통한 실시간 명령 수신 확인.
  Step 4: 상태 머신(State Machine) 기반 인터랙션 구현
    - '명령 수신' -> '확인 질문' -> '긍정 답변 시 실행'의 루프 완성.

5. 메모리 관리 팁
  - Isaac Sim 구동 시 `nvidas-smi`로 VRAM 상시 모니터링.
  - LLM 모델이 너무 무거울 경우 3.8GB 수준인 Phi-3.5 Mini로 교체 검토.
  - Whisper STT 구동 시 `compute_type="int8"` 옵션을 사용하여 VRAM 절약.
